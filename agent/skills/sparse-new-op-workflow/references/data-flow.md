# 数据流说明

## 文档存储

所有开发中间文档存放在 `.agent/dev-docs/{op_name}/`：

```
.agent/dev-docs/{op_name}/
├── LOG.md
├── 2.0.1-开发环境.md
├── 1.1-参考资料清单.md
├── 1.2-需求分析.md
├── 1.3.A-开发方案设计.md
├── 1.3.B-测试方案设计.md
├── 1.4.A-开发方案评审.md
├── 1.4.B-测试方案评审.md
├── 2.1.2-汇合联调报告.md
├── 2.1.3-测试验收报告.md
├── 2.2.2-汇合联调报告.md
├── 2.2.3-测试验收报告.md
├── 3.1-代码检视报告.md
├── 3.2-性能报告.md
├── 4.1.1-审查报告.md
├── 4.1.2-编译测试报告.md
├── 4.2-代码检视报告.md
├── 4.3-Issue.md
├── 4.3-上库PR模板.md
├── CP1.1.A.json / CP1.1.B.json / CP1.2.json ~ CP4.3.json
└── issues/
    └── issue_{YYYYMMDD}_{关键词}.md
```

## 各阶段数据流

### 阶段一：设计

```
用户需求
    │
    └─→ [1.1.A 资料准备] → 工作区目录(临时名) + LOG.md + 1.1-参考资料清单.md
            │
            └─→ [1.1.S 总结] → CP1.1.A.json
                    │
                    └─→ ⛔ CP1.1.A（算子名/数据类型/目标芯片）

CP1.1.A 确认算子名
    │
    └─→ [1.1.B 环境准备] → 2.0.1-开发环境.md + git 分支

CP1.1.A结论 + 1.1-参考资料清单.md
    │
    └─→ [1.1.S2 总结] → CP1.1.B.json
            │
            └─→ ⛔ CP1.1.B（精度标准/编程模型）

CP1.1结论 + 1.1-参考资料清单.md
    │
    └─→ [1.2 需求分析] → 1.2-需求分析.md + CP1.2.json
            │
            └─→ ⛔ CP1.2（用户审批）

1.2-需求分析.md
    ├─→ [1.3.A ⇄ 1.4.A 循环 ≤3] → 1.3.A + 1.4.A（开发侧）
    └─→ [1.3.B ⇄ 1.4.B 循环 ≤3] → 1.3.B + 1.4.B（测试侧）
```

### 阶段二：开发

```
1.3.A-开发方案设计.md → [2.1.1.A 算子开发] → 算子代码    ┐
1.3.B-测试方案设计.md → [2.1.1.B 测试开发] → 用例代码    ┤ 并行
                                                          │
算子代码 + 用例代码                                        │
    └─→ [2.1.2 汇合联调] → [2.1.3 测试验收] → ⚪ CP2.1    │
                                                          │
（迭代二结构相同）                                         │
```

### 阶段三：验收

```
git diff + OAT 报告 + 全部变更文件
    └─→ [3.1 代码检视 ⇄ developer 修复 循环 ≤3] → 3.1-代码检视报告.md
            │
            └─→ [3.2 性能验收] → 3.2-性能报告.md
                    │
                    └─→ ⛔ CP3.2

CP3.2 问卷结果（用户选择「精简为 1 条」时）
    └─→ [3.3 大 shape 精简] → 精简后的 CSV + ST 通过
```

### 阶段四：上库

```
    └─→ [4.1 编写文档] → README.md
            │
            └─→ [4.1.1 README 内容审查] → 4.1.1-审查报告.md
                    │                        （不通过 → 打回 4.1，≤2 次）
                    └─→ [4.1.2 README 编译测试] → 4.1.2-编译测试报告.md
                            │                        （不通过 → 打回 4.1，≤2 次）
                            └─→ [4.2 代码检视] → 4.2-代码检视报告.md
    └─→ [4.3 开发总结] → CP4.3.json + Issue + PR 模板 + LOG
            └─→ ⛔ CP4.3（squash commit）
```
