## 产出要求

1. 生成 `spec.yaml`（推荐用 `ops-spec-gen` 的向导 / `generate_spec.py`）。
2. 通过 9-stage 校验：`python3 ops/ops-spec-gen/scripts/validate_spec.py <spec.yaml>` **全 PASS**（stage 9 SKIP 视为通过）。
3. `spec.yaml` 各字段与 1.1 需求文档一致（dtype / shape / 平台 / 容差可追溯）。

## 稀疏算子注意事项

- Spec 中需明确标注 API 体系（Generic / Legacy），与 1.1 需求分析中的 cuSPARSE 对标一致
- 稀疏格式（CSR/COO/CSC）作为算子规格的一部分，需在 spec 中体现
- 描述符类型（SpMatDescr/DnVecDescr/DnMatDescr）根据 API 体系选择
- Legacy API 的精度前缀（S/D/C/Z）需在 spec 中标注

## 字段职责边界（勿越权）

- `spec.yaml` 只承载已锁定的结构化 **L0 语义契约**；需求背景、接口自然语言说明、性能目标仍由需求文档承载。
- `test_matrix` 已移出 spec，由 `ascendc-st-design` skill 管理；容差引用 `ops-precision-standard`。
- 下游方案/测试设计遇 L0 字段冲突时**停止并报告**，不得从需求正文重新解释后覆盖。
