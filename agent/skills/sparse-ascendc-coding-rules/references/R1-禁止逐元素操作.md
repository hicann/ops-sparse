# R1: 禁止逐元素操作

## 说明

`SetValue`/`GetValue` 和逐元素的 `DataCopy`/`DataCopyPad` 本身并无问题，但被禁止使用是因为它们会诱导逐元素串行操作，使 Kernel 退化为 CPU 代码，丧失 NPU 的矢量并行和 DMA 批量搬移能力。

## 错误示例

```cpp
// 逐元素操作 → 串行执行，性能极差
for (int i = 0; i < n; i++) {
    DataCopy(dst[i], src[i], 1);
}
```

## 正确示例

```cpp
// 批量搬移 → 利用 DMA 并行搬移
DataCopy(dst, src, n);
```
